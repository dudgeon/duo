⚠️ Class B perf — inverted finding. Synthetic F1 on a fresh claude -p --model sonnet, comparing inline (Sonnet → Bash(duo *)) vs subagent (Sonnet → Task(duo)): subagent path was ~2× cost ($0.17 vs $0.08) and ~2× wall-clock (65s vs 36s). Cause: Claude Code already routes mechanical tool execution to Haiku regardless of --model, so the subagent stacks a SECOND Haiku context on top of the existing fast-tier Haiku — doubling the Haiku-side cost. The PRD's "~85% orchestrator-token reduction" was framed against a mental model where the top-level Sonnet processes CLI dumps directly; that isn't how Claude Code distributes tokens. The subagent's value is real but qualitative — bounded per-task context, specialized prompt, predictable contract — not the per-task dollar win on a cold-cache synthetic. Re-measurement with cumulative-context methodology filed as FOLLOWUP-003 in tasks.md.

FOLLOWUP-002 (low priority). When a user has a Bash allowlist tight enough to deny the guard's compound [ … ] && echo … command, the guard silently fails and the agent proceeds. Permissive Bash (the realistic default) works correctly. Fix: tighten the agent prompt to refuse-on-check-denied. Filed in tasks.md.

End-user first launch: Stage 18's installer copies the bundled file after the consent sheet. End-user app update: hash bundled vs installed; silent overwrite if unchanged, prompt-with-diff if user-modified.

Refinement bundle for the file navigator: selection model, ambient signal density (active CWD / open files), file-mutation actions, and a "new Claude here" affordance. Items 1, 6, and 7 share the row-interaction surface and ship well together — single-click selects, double-click opens, right-click acts, hover reveals action chip. Item 3 refines an existing Stage 10 follow-up ("highlight files open in WorkingPane tabs") with the open-vs-active distinction.

  • 1. Double-click to open, single-click to select. Today single-click opens the file in the right pane, which means there's no way to select a row without opening it. Blocks any context-menu-driven action (item 6) and accidentally promotes-on-stray-click. Recommend Finder / VS Code parity: single-click selects (visible row state), double-click opens, ↵ on selected row also opens, ⎋ clears selection. Tension with Stage 10 PRD's "Clicking a .md file opens it" — that line needs an amendment when this lands.
  • 2. Highlight active terminal's CWD in the tree. When a folder row matches the focused terminal's CWD, render a subtle folder-accent (distinct from selection — this is ambient signal, not interaction state). Plumbing: navigator subscribes to the focused PTY's cwd field that Stage 10 already tracks for the breadcrumb / pending-CWD path.
  • 3. Distinguish open-but-inactive from active in the file row highlight. Refines the existing Stage 10 follow-up "highlight files open in WorkingPane tabs" with a second state: open = subtle accent, active = stronger (filled chip or border-left). Three signal layers in total — selected (1), open / active (3), CWD (2). Define them so they stack without becoming visual noise.
  • 4. Verify "new terminal" defaults CWD to current navigator view. Spike (~30 min): does opening a fresh terminal tab today inherit the navigator's current folder, or only the explicit "open terminal here" gesture? Stage 10 PRD calls out "here" but is silent on the default. If it doesn't inherit, file as a bug; if it does, document and close.
  • 5. Bug: deleted files linger in navigator. When a file is removed on disk (terminal rm, Finder, agent), the tree row stays until a full reload. chokidar should fire unlink — either the watcher isn't subscribed for it in the navigator data path, or the tree-mutation reducer ignores the event. Filed as BUG-007 in tasks.md; sweep should also cover rename / move (likely the same branch is dropping unlinkDir → addDir rename pairs).
  • 6. Right-click → Delete / Rename in navigator. Context menu actions on file / folder rows. Recommend: delete = shell.trashItem (macOS-native trash, undoable from Finder) with a single-click confirm; rename = inline contenteditable on the row label, ⎋ cancels, ↵ commits. CLI parity per CLAUDE.md §4: duo file rename <old> <new> + duo file trash <path>.
  • 7. Hover-action: "new Claude here" on folder rows. Hovering a folder reveals a tiny clawd-icon button on the row's right edge → opens a new terminal tab with CWD = that folder AND auto-launches claude in the fresh PTY. Distinct from the existing "open terminal here" (bare shell). Open questions: (a) inherit Claude flags from user config; (b) what happens if claude isn't on PATH (silent fail, error toast, fallback to bare shell?); (c) does the new-tab strip animate the launch or run it silently after focus. CLI parity could reuse the backlog's duo tab --cmd (15-family) as duo new-tab --cwd <path> --cmd "claude" rather than minting a separate verb.

Cross-refs: Stage 10 follow-ups (tree-expand persistence + open-tab highlight — item 3 refines the latter); backlog 15-family duo tab --cmd (item 7's CLI parity). When this lands, Stage 10 PRD's "clicking a .md opens it" line gets amended for item 1's double-click contract.

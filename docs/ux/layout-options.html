Duo — Layout Options

Ten candidate layouts for the main window. The variables are: where the terminal sits, whether the working pane is tabbed or mode-toggled, and how the two side panels (file browser + agent tools) are exposed. Review side by side, then we update the brief.

Terminal (agent) Working pane — browser / file viewer / markdown editor File browser Agent tools / skills Drawer (hidden by default)

01Cockpit

Your proposed layout

Two persistent sidebars flank a primary split: working pane left (tabbed, polymorphic), terminal right. The agent is the dominant action surface; the working pane is what it operates on.

06Stage

Content hero

The working pane takes the top two-thirds of the window; the terminal is a full-width strip beneath. Sidebars are drawers triggered by hotkeys.

Workingdominant
Terminal

10Zen

Palette-driven

Two-pane main window — working left, terminal right — and that's it. File browser and agent tools are palette overlays (Cmd-P, Cmd-K) rather than persistent panels.

Working
Terminal

Best for: keyboard-native users who dislike permanent sidebars and prefer invoking panels on demand.

Pros

  • Maximum screen area for content
  • Cleanest, most intentional aesthetic
  • Palette scales well as features grow

Cons

  • Worst discoverability — new users are lost
  • File browser as palette loses spatial navigation
  • Skills viewer in a palette is awkward to browse

How to read the wireframes. Tab strips (the little rectangles along the top of a region) indicate that surface is tabbed. Dashed outlines indicate drawers — panels hidden by default, invoked by hotkey or hover. All layouts assume the working pane is polymorphic (browser / markdown editor / file viewer) with per-tab mode.
